# Learning Path: Executive

**Target Audience:** C-suite executives, VP-level leaders, board members
**Total Time:** ~70 minutes
**Goal:** Understand AI transformation strategy, organizational requirements, and decision frameworks for AI adoption

---

## Why This Path

As an executive, you need to:
- Understand the strategic landscape of AI transformation (not just technical details)
- Make informed build-vs-buy decisions
- Assess organizational readiness and required team structure
- Evaluate vendor claims and solution proposals
- Set realistic expectations for timelines and ROI

This path focuses on **strategic understanding** without implementation details.

### Module 2: F2 - Organization Structure (18 min)

**Why take this:** Understand WHO makes transformation happen

**What you'll learn:**
- Four key roles: Chief AI Officer, AI Champions, Agent Managers, AI Agent Developers
- Why the Agent Manager role is critical (domain experts who document processes)
- How Agent Manager + Developer partnership enables rapid iteration (Level 4 defining structure)
- Real implementation stories: HR, Sales, Operations showing 20 iteration patterns over 2-3 weeks
- Transformation process: 3 phases, realistic timelines, budget requirements

### Module 3: S1 - Choosing AI Technology (18 min)

**Why take this:** Make informed build-vs-buy decisions

**What you'll learn:**
- Four AI approaches: Machine Learning, Custom Agentic AI, Existing Tools (with/without data access), No AI
- When to use each approach with real examples
- Why context engineering gives custom builds advanced control (model quality, retrieval logic, interfaces, capabilities)
- Three Agentic solution categories with ROI ranges ($15K-$246K/year)
- Decision framework: When existing tools are sufficient vs when custom builds are necessary

**Key takeaway:** Many existing tools NOW can access your data (Notion AI, Copilot) - but custom gives you control over models, capabilities, interfaces, and retrieval logic that existing tools can't match.

**Strategic value:** You'll know exactly when to build custom vs use existing tools, avoiding both over-investment and missed opportunities.

## Common Executive Questions

**Q: Should I hire an AI consultant or build internal team?**

A: Build internal team (Agent Managers + Developers). Consultants can't iterate rapidly (1 cycle/week vs 1-3/day). You'll hit the Level 3 trap - 5+ months â†’ 65-75% reliability â†’ abandoned. See F1 for iteration economics.

**Q: How much should I budget for first AI transformation initiative?**

A: $100K-$500K for custom build depending on complexity. Includes: 30% technical build, 35% process documentation, 35% context engineering. Timeline: 2-4 months. See F2 for team sizing and S1 for solution categories with ROI ranges.

**Q: Can we just use Notion AI / Microsoft Copilot / Salesforce Einstein?**

A: Depends on your needs. These tools CAN access your data BUT are limited by:
- Vendor's model choice (often cheaper LLMs)
- Basic capabilities (simple Q&A, no reflection/verification)
- Their interface (can't deploy as Slackbot if team lives in Slack)
- Their roadmap (not yours)

Use S1 decision framework: Check the four limitation factors (interface, model quality, capabilities, features). If all acceptable â†’ use existing tool. If any insufficient â†’ consider custom build.

**Q: What's a realistic timeline to reach 95%+ reliability?**

A: With Level 4 structure (internal team, rapid iteration):
- 2-3 weeks to production-ready (20 iterations at 1-3/day)
- Start at 70-85% accuracy
- Reach 95%+ through feedback loops

With Level 3 approach (external agency, slow iteration):
- 6 months â†’ Still not production-ready â†’ Often abandoned
- Stuck at 70% because only 12-24 total iterations possible

See F1 for iteration economics breakdown.

**Q: Do I need AI experts or can I promote from within?**

A: Promote domain experts to Agent Managers. They know the processes deeply - they just need to learn to document for AI consumption (not a technical skill). Hire 1-2 Developers who understand AI architecture. See F2 for role definitions and team composition.

**Q: How do I know if a vendor solution is truly "agentic"?**

A: Use S2 component checklist:
- âœ“ LLM (which model? Latest generation?)
- âœ“ Tools (what can it actually DO beyond chat?)
- âœ“ Memory (maintains context across interactions?)
- âœ“ Autonomy (breaks down goals, self-corrects?)

Also ask:
- How do we iterate when outputs aren't right? (daily or monthly?)
- Who controls iteration - us or you?
- What's the path to 95%+ reliability? (need 20 iterations over 2-3 weeks)

Red flags: "AI-powered" with no clear tools, "automation" that's manual execution, "custom" with slow iteration cycles.
